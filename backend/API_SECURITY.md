## 1. 权威与访问控制

- 服务端授权是权威裁决；客户端检查仅用于 UX。
- 认证和授权变更需要 Level 2 或更高等级。
- 不得绕过目标项目既有的身份、会话、令牌、策略或权限模型。
- 权限变更必须包含负向路径行为，并在存在角色、租户、所有权范围或等价访问维度时识别受影响范围。

---

## 2. 边界校验

- 必须在 API 边界校验外部输入。
- 必须按照目标项目约定校验标识符、分页、排序、过滤、上传内容、命令载荷和集成回调。
- 必须在业务逻辑、持久化或下游调用之前拒绝无效或未授权输入。

---

## 3. 响应安全

- 不得在响应中暴露密钥、内部堆栈跟踪或敏感标识符。
- 错误响应必须具备可操作性，同时不得泄露内部细节或敏感数据。
- 状态变更操作必须保留目标项目的防护机制。
